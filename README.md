# Book Voicing

Локальный конвейер для озвучивания FB2-книг на Windows 11 с помощью Chatterbox Multilingual и NVIDIA GPU.

## Возможности

- извлечение названия, авторов и вложенных глав из FB2;
- очистка и разбиение текста на небольшие фрагменты;
- одна загрузка модели Chatterbox на весь запуск;
- клонирование разрешённого пользователю голоса по WAV-референсу;
- атомарное сохранение WAV без повреждённых файлов после сбоя;
- продолжение работы с пропуском уже готовых фрагментов;
- автоматическая сборка M4B с AAC-аудио и маркерами глав;
- локальные ffmpeg и ffprobe из npm, без ручной настройки PATH.

## Требования

- Windows 11;
- NVIDIA RTX 3060 12 GB или аналогичная карта;
- актуальный драйвер NVIDIA;
- Python 3.10-3.12 x64;
- Node.js 20 или новее;
- Git.

## Установка

Откройте PowerShell в каталоге проекта:

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\scripts\setup-windows.ps1
```

Скрипт создаёт изолированное `.venv`, устанавливает Chatterbox, совместимый PyTorch с CUDA и Node.js-зависимости. Системный Python не изменяется.

Проверка окружения:

```powershell
.\.venv\Scripts\python.exe python\check_cuda.py
npm run check
```

Ожидаемый результат содержит `CUDA available: True`, название видеокарты и `Chatterbox import: OK`.

## Подготовка голоса

Положите чистую запись длительностью примерно 10-20 секунд в WAV-файл, например `voices/reader.wav`.

Рекомендации:

- один говорящий;
- без музыки, эха и сильного шумоподавления;
- спокойная естественная речь;
- русский язык для русской книги;
- используйте только голос, на который у вас есть разрешение.

## Первый тест

Положите книгу в `books/book.fb2` и выполните:

```powershell
npm run voice -- --input books\book.fb2 --voice voices\reader.wav --limit 3
```

Будут созданы первые три WAV-фрагмента и тестовый M4B в `output/<название-книги>/`.

Тест только WAV без сборки M4B:

```powershell
npm run voice -- --input books\book.fb2 --voice voices\reader.wav --limit 3 --no-m4b
```

Полная книга:

```powershell
npm run voice -- --input books\book.fb2 --voice voices\reader.wav
```

Повторный запуск продолжит генерацию, пропустив существующие WAV. Для принудительной перегенерации добавьте `--overwrite`.

## Параметры

```text
--input         путь к FB2, обязателен
--voice         путь к WAV-референсу
--output        каталог результата, по умолчанию output
--chunk-size    максимум символов в фрагменте, по умолчанию 850
--limit         обработать только первые N фрагментов
--overwrite     перегенерировать существующие WAV
--exaggeration  выразительность Chatterbox, по умолчанию 0.5
--cfg-weight    сила следования голосу, по умолчанию 0.5
--bitrate       битрейт итогового AAC, по умолчанию 96k
--no-m4b        не собирать итоговую аудиокнигу
```

Для спокойного книжного чтения:

```powershell
npm run voice -- --input books\book.fb2 --voice voices\reader.wav --limit 5 --exaggeration 0.35 --cfg-weight 0.4
```

## Результат

```text
output/<книга>/
├── chunks/              текстовые и WAV-фрагменты
├── job.json             последнее задание генерации
├── manifest.json        главы и порядок фрагментов
└── <книга>.m4b          итоговая аудиокнига
```

Временный каталог `.work` удаляется после успешной сборки. При ошибке его можно удалить вручную и повторить команду.

## Ограничения текущей версии

- поддерживается только FB2;
- обложка пока не внедряется в M4B;
- GUI ещё отсутствует;
- сноски и нестандартная структура отдельных FB2 могут потребовать дополнительной нормализации.

Первый запуск Chatterbox скачивает модель. Если не хватает видеопамяти, уменьшите `--chunk-size` до 500-650 и закройте приложения, активно использующие GPU.
