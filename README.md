# Book Voicing

Локальный конвейер для озвучивания FB2-книг на Windows 11 с помощью Chatterbox Multilingual и NVIDIA GPU.

## Что уже умеет

- извлекать метаданные и главы из FB2;
- очищать текст от служебной разметки;
- делить главы на небольшие фрагменты;
- вызывать Python-адаптер Chatterbox из Node.js;
- сохранять WAV-фрагменты по главам;
- продолжать работу после прерывания, пропуская уже готовые файлы;
- клонировать голос по референсному WAV-файлу.

На первом этапе проект намеренно не склеивает результат в M4B. Сначала стоит проверить качество русского голоса и подобрать параметры генерации. Склейку и главы M4B добавим после успешного теста.

## Требования

- Windows 11;
- NVIDIA RTX 3060 12 GB или аналогичная карта;
- актуальный драйвер NVIDIA;
- Python 3.11 x64;
- Node.js 20 или новее;
- Git.

## Быстрая установка

Откройте PowerShell в каталоге проекта:

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\scripts\setup-windows.ps1
```

Скрипт создаст `.venv`, установит Node.js-зависимости, PyTorch с CUDA и `chatterbox-tts`.

Проверка CUDA:

```powershell
.\.venv\Scripts\python.exe python\check_cuda.py
```

Ожидаемый результат содержит `CUDA available: True` и название видеокарты.

## Подготовка голоса

Положите чистую запись русского голоса длительностью примерно 10-20 секунд в WAV-файл, например:

```text
voices/reader.wav
```

Желательно:

- один говорящий;
- без музыки, эха и шумоподавления;
- обычная спокойная речь;
- тот же язык, на котором будет книга;
- использовать только голос, на который у вас есть разрешение.

## Первый тест

Положите книгу в `books/book.fb2` и выполните:

```powershell
npm run voice -- --input books\book.fb2 --voice voices\reader.wav --limit 3
```

`--limit 3` озвучит только первые три фрагмента. Результат появится в `output/<название-книги>/`.

Для всей книги:

```powershell
npm run voice -- --input books\book.fb2 --voice voices\reader.wav
```

Без клонирования голоса:

```powershell
npm run voice -- --input books\book.fb2 --limit 3
```

## Полезные параметры

```text
--input        путь к FB2, обязателен
--voice        путь к WAV-референсу, необязателен
--output       каталог результатов, по умолчанию output
--chunk-size   приблизительный максимум символов, по умолчанию 850
--limit        ограничить число генерируемых фрагментов
--overwrite    перегенерировать существующие WAV
--exaggeration выразительность, по умолчанию 0.5
--cfg-weight   сила следования голосу, по умолчанию 0.5
```

Для более спокойного книжного чтения часто стоит попробовать:

```powershell
npm run voice -- --input books\book.fb2 --voice voices\reader.wav --limit 5 --exaggeration 0.35 --cfg-weight 0.4
```

## Структура

```text
src/                 Node.js: CLI, FB2 и оркестрация
python/              Python: Chatterbox и проверка CUDA
scripts/             установка под Windows
books/               входные книги, не коммитятся
voices/              голосовые референсы, не коммитятся
output/              результат, не коммитится
```

## Замечания

Chatterbox Multilingual загружает модель при первом запуске, поэтому первый старт дольше последующих. Для русского используется `language_id="ru"`. Проект использует Multilingual V3, рекомендованную разработчиками Chatterbox.

Если возникает ошибка памяти, уменьшите `--chunk-size` до 500-650 и закройте приложения, использующие GPU.
